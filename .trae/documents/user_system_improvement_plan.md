# RealWorld Axum API - 用户系统改进计划

## [x] 任务 1: 恢复密码验证逻辑
- **优先级**: P0
- **依赖关系**: 无
- **描述**:
  - 取消 `auth.rs` 中密码验证代码的注释
  - 确保 `verify_password` 函数正确实现
  - 测试登录功能，确保密码验证正常工作
- **成功标准**:
  - 登录接口能够正确验证用户密码
  - 密码错误时返回 401 Unauthorized 状态码
- **测试要求**:
  - `programmatic` TR-1.1: 正确密码登录返回 200 OK
  - `programmatic` TR-1.2: 错误密码登录返回 401 Unauthorized
- **备注**:
  - 确保密码验证逻辑与密码哈希逻辑匹配

## [x] 任务 3: 增强安全性 - 实现速率限制
- **优先级**: P1
- **依赖关系**: 无
- **描述**:
  - 添加速率限制中间件，限制登录尝试次数
  - 配置合理的速率限制规则（如 1 分钟内最多 5 次尝试）
- **成功标准**:
  - 登录接口在短时间内多次尝试后返回 429 Too Many Requests
  - 速率限制规则能够有效防止暴力攻击
- **测试要求**:
  - `programmatic` TR-3.1: 1 分钟内连续 6 次登录尝试后返回 429
  - `programmatic` TR-3.2: 等待一段时间后登录尝试恢复正常
- **备注**:
  - 考虑使用 Redis 存储速率限制计数器

- 错误处理更加完善，能够提供清晰的错误信息
- 代码质量得到提高，更加易于维护和扩展
- 功能更加丰富，能够满足用户的各种需求

这些改进将使 RealWorld Axum API 成为一个更加成熟、可靠的用户系统，为用户提供更好的体验。
